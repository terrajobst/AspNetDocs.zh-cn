## <a name="log-for-insight"></a><span data-ttu-id="3e286-175">了解日志</span><span class="sxs-lookup"><span data-stu-id="3e286-175">Log for insight</span></span>

<span data-ttu-id="3e286-176">遥测包是一个很好的第一步，但你仍需要检测自己的代码。</span><span class="sxs-lookup"><span data-stu-id="3e286-176">A telemetry package is a good first step, but you still have to instrument your own code.</span></span> <span data-ttu-id="3e286-177">遥测服务会在出现问题时通知你，并告诉你客户遇到的问题，但它可能不会向你深入了解代码中的内容。</span><span class="sxs-lookup"><span data-stu-id="3e286-177">The telemetry service tells you when there's a problem and tells you what the customers are experiencing, but it may not give you a lot of insight into what's going on in your code.</span></span>

### <a name="log-in-production"></a><span data-ttu-id="3e286-182">记录生产</span><span class="sxs-lookup"><span data-stu-id="3e286-182">Log in production</span></span>

<span data-ttu-id="3e286-183">许多人只是在出现问题时才会在生产环境中启用跟踪。</span><span class="sxs-lookup"><span data-stu-id="3e286-183">A lot of people turn on tracing in production only when there's a problem and they want to debug.</span></span> <span data-ttu-id="3e286-184">这可能会导致你发现问题的时间与获取有用的故障排除信息的时间之间存在巨大的延迟。</span><span class="sxs-lookup"><span data-stu-id="3e286-184">This can introduce a substantial delay between the time you're aware of a problem and the time you get useful troubleshooting information about it.</span></span> <span data-ttu-id="3e286-185">而且，您获取的信息对于间歇错误可能并不有用。</span><span class="sxs-lookup"><span data-stu-id="3e286-185">And the information you get might not be helpful for intermittent errors.</span></span>

<span data-ttu-id="3e286-186">对于存储成本较低的云环境，我们建议您始终在生产中保留日志记录。</span><span class="sxs-lookup"><span data-stu-id="3e286-186">What we recommend in the cloud environment where storage is cheap is that you always leave logging on in production.</span></span> <span data-ttu-id="3e286-187">这样，当发生错误时，您已经记录了这些错误，您可以使用历史数据来帮助您分析随时间推移或在不同时间定期发生的问题。</span><span class="sxs-lookup"><span data-stu-id="3e286-187">That way when errors happen you already have them logged, and you have historical data that can help you analyze issues that develop over time or happen regularly at different times.</span></span> <span data-ttu-id="3e286-188">你可以自动执行清除过程以删除旧的日志，但你可能会发现设置此类进程的开销要比保留日志的开销更高。</span><span class="sxs-lookup"><span data-stu-id="3e286-188">You could automate a purge process to delete old logs, but you might find that it's more expensive to set up such a process than it is to keep the logs.</span></span>
